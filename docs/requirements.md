# エンジニア向けスキルシート作成アプリ 要件定義書

## 1. 概要

### 1.1 コンセプト
「エンジニアのスキルシートを統一し、誰もが手軽にスキルシートを編集してスキルをアピールできるようにする」

### 1.2 背景・課題
- エンジニア毎にスキルシートのフォーマットが異なる
- 会社や営業によって新しくスキルシートを作らされるのが面倒

### 1.3 対象ユーザー
- エンジニア（主にフリーランスエンジニア想定）

### 1.4 プラットフォーム
- Webアプリ限定
- モバイル対応は現状なし（レスポンシブは広告表示のみ対応）

---

## 2. 機能要件

### 2.1 初回リリーススコープ
- スキルシート作成機能のみ
- 契約書作成機能は将来対応（Coming Soon表示）

### 2.2 トップページ

4つのカードを配置：
| カード | 状態 |
|--------|------|
| 新しくスキルシートを作成 | 有効 |
| 以前のデータを読み込みスキルシートを作成 | 有効 |
| 新しく契約書を作成 | 非活性（Coming Soon） |
| 以前のデータを読み込み契約書を作成 | 非活性（Coming Soon） |

### 2.3 入力ステップ（全10ステップ）

| ステップ | 画面名 | 内容 |
|---------|--------|------|
| 1 | プロフィール | 氏名、ふりがな、生年月日、年齢、性別、資格、得意分野 |
| 2 | 住所 | 都道府県、市区町村、最寄駅 |
| 3 | 連絡先 | 電話番号、メールアドレス |
| 4 | 職務要約 | 100〜150文字推奨、200文字上限 |
| 5 | スキル | スキル名、経験年数、習熟度・説明（複数追加可） |
| 6 | 職務経歴 | プロジェクト詳細（複数追加可、新しい順） |
| 7 | 自己PR | 1000文字以内 |
| 8 | 作成日 | 自動入力、編集可能 |
| 9 | 確認 | PDFプレビュー表示 |
| 10 | 完了 | ダウンロード、保存、アフィリエイトリンク |

## 3. 入力項目詳細

### 3.1 プロフィール（ステップ1）

| 項目 | 形式 | 必須 | 備考 |
|------|------|------|------|
| 姓 | テキスト | ○ | |
| 名 | テキスト | ○ | |
| セイ（ふりがな） | テキスト | ○ | カタカナ |
| メイ（ふりがな） | テキスト | ○ | カタカナ |
| 生年月日 | 年月日選択 | ○ | YYYY年MM月DD日 |
| 年齢 | 自動計算 | - | 生年月日から自動計算、表示のみ |
| 性別 | ラジオボタン | ○ | 男性/女性/その他/回答しない |
| 資格 | テキストエリア | ○ | 複数羅列形式 |
| 得意分野 | テキスト | ○ | プレースホルダー「例:Webアプリケーション設計、要件定義、Webアプリケーション開発、Webデザイン作成」 |

### 3.2 住所（ステップ2）

| 項目 | 形式 | 必須 | 備考 |
|------|------|------|------|
| 郵便番号 | テキスト | ○ | ハイフンあり、APIで住所自動入力 |
| 都道府県 | 自動入力/選択 | ○ | 郵便番号から自動入力 |
| 市区町村 | 自動入力/テキスト | ○ | 郵便番号から自動入力、不足分は手動入力 |
| 最寄駅 | テキスト | ○ | プレースホルダー「例:JR渋谷駅」 |

### 3.3 連絡先（ステップ3）

| 項目 | 形式 | 必須 | 備考 |
|------|------|------|------|
| 電話番号 | テキスト | ○ | ハイフンあり形式 |
| メールアドレス | テキスト | ○ | 形式チェックあり（@含む等） |

### 3.4 職務要約（ステップ4）

| 項目 | 形式 | 必須 | 備考 |
|------|------|------|------|
| 職務要約 | テキストエリア | ○ | 200文字上限 |

**表示・バリデーション:**
- ガイド文言: 「職務要約を100〜150文字程度でわかりやすくまとめましょう」
- 200文字超過時: 入力枠が赤くなり、ガイド文言を再表示
- 箇条書き・改行可能

### 3.5 スキル（ステップ5）

カード形式で複数追加可能（上限なし）

| 項目 | 形式 | 必須 | 備考 |
|------|------|------|------|
| スキルや資格の名称 | テキスト | ○ | 例: Javascript, Webデザイン |
| 経験年数 | 数値 | ○ | 「○年」形式 |
| 習熟度・説明 | テキストエリア | ○ | フレームワーク等の詳細を記述 |

**操作:**
- 「+」ボタンでカード追加
- 「削除」ボタンでカード削除
- 「OK」ボタンで入力確定

### 3.6 職務経歴（ステップ6）

カード形式で複数追加可能（上限なし、新しい順で表示）

| 項目 | 形式 | 必須 | 備考 |
|------|------|------|------|
| 開始年月 | 年月選択 | ○ | YYYY年MM月から |
| 終了年月 | 年月選択 | ○ | YYYY年MM月まで |
| 在職中 | チェックボックス | - | チェック時は終了年月を「現在」扱い |
| 稼働期間 | 自動計算 | - | 期間から自動計算（例: 9ヶ月） |
| プロジェクト名 | テキスト | ○ | |
| 業務概要 | テキストエリア | ○ | 担当業務、習得スキル、コメント等 |
| 役割 | テキスト | ○ | 例: PM、SE、PG |
| 規模 | テキスト | ○ | 例: チーム6名、全体80名 |
| 使用言語 | テキスト | ○ | 例: PHP, SQL |
| FW・MW・ツール等 | テキスト | ○ | 例: VMware, Subversion |

**操作:**
- 最上部の「+」ボタンで新規カード追加（新しいものが上に来る）
- 「削除」ボタンでカード削除

### 3.7 自己PR（ステップ7）

| 項目 | 形式 | 必須 | 備考 |
|------|------|------|------|
| 自己PR | テキストエリア | ○ | 1000文字以内 |

**表示:**
- ガイド文言: 「自己PR文を1000文字以内で入力してください」
- 箇条書き・改行可能

### 3.8 作成日（ステップ8）

| 項目 | 形式 | 必須 | 備考 |
|------|------|------|------|
| 作成日 | 日付選択 | ○ | 今日の日付を自動入力、編集可能 |

### 3.9 確認（ステップ9）

- 実際のPDF形式でプレビューを表示
- 入力内容の最終確認

### 3.10 完了（ステップ10）

| 要素 | 説明 |
|------|------|
| 「PDFでダウンロード」ボタン | 押下で動画広告表示→終了後ダウンロード |
| 「入力情報をファイルで保存」ボタン | 押下で動画広告表示→終了後JSONダウンロード |
| 「フリーランスの案件をお探しの方はこちら」ボタン | アフィリエイトリンク |
| 「転職をお考えの方はこちら」ボタン | アフィリエイトリンク |

## 4. 画面共通仕様

### 4.1 ナビゲーション

| 要素 | 説明 |
|------|------|
| ステップインジケーター | 10ステップを表示、現在位置をハイライト |
| 「前へ」ボタン | 前のステップに戻る |
| 「次へ」ボタン | 次のステップに進む（バリデーション通過時） |
| 「プレビュー」ボタン | 画面下部に常時配置、入力途中でもPDFプレビュー可能 |

### 4.2 プレビュー機能

- 別画面に遷移してPDFプレビュー表示
- 「入力画面に戻る」ボタンで遷移前の画面に戻る

### 4.3 ヘッダー

- アプリ名/ロゴ（デザイン共有時に決定）

### 4.4 フッター

- 利用規約へのリンク
- プライバシーポリシーへのリンク

### 4.5 ブラウザ離脱防止

- ブラウザ/タブを閉じようとした際にアラートを表示
- ブラウザを閉じた場合、入力データは消失してOK

## 5. バリデーション

### 5.1 必須チェック

- 全項目必須

### 5.2 形式チェック

| 項目 | チェック内容 |
|------|-------------|
| メールアドレス | @を含む形式チェック |
| 電話番号 | ハイフンあり形式 |
| 職務要約 | 200文字以下 |
| 自己PR | 1000文字以下 |

### 5.3 エラー表示

- エラー時は入力枠を赤く表示
- エラーメッセージを赤文字で表示

---

## 6. データ保存・読み込み

### 6.1 保存形式

- JSON形式
- ファイル名: 自動生成（例: `skillsheet_20260302.json`）

### 6.2 読み込み

- トップページ「以前のデータを読み込みスキルシートを作成」から読み込み
- ファイル選択ダイアログでJSONファイルを選択

### 6.3 エラーハンドリング

| ケース | エラーメッセージ |
|--------|----------------|
| 不正なJSONファイル | 「不正なファイルなので読み込めません」 |
| 古いバージョン/破損ファイル | 「ファイルのバージョンが古いかファイルが破損しているため読み込めません」 |
| ファイル読み込み失敗 | 赤文字でエラー表示 |

## 8. 広告・マネタイズ

### 8.1 バナー広告

| 配置 | 表示条件 |
|------|----------|
| 左側 | 990px超で表示 |
| 右側 | 720px超で表示 |
| 下部 | 常時表示 |

- 全画面で常時表示
- Google AdSense使用

### 8.2 動画広告

| タイミング | 動作 |
|------------|------|
| 「PDFでダウンロード」ボタン押下時 | 動画広告表示→終了後ダウンロード開始 |
| 「入力情報をファイルで保存」ボタン押下時 | 動画広告表示→終了後ダウンロード開始 |

- 5秒後スキップ可能、または20秒程度の動画
- Google AdSense使用

### 8.3 アフィリエイト

| リンク | 文言 |
|--------|------|
| フリーランスエージェント | 「フリーランスの案件をお探しの方はこちら」 |
| 転職エージェント | 「転職をお考えの方はこちら」 |

- リンク先は後から差し替え可能な設計にする
- 完了画面に配置
